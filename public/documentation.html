Movies Endpoints

Get All Movies

This endpoint returns a full list of movies in the database.

  • URL: /movies
  • Method: GET
  • Query Parameters: N/A
  • Expected Body Format: N/A
  • Response Format: JSON

Get Movie by Title

This endpoint returns details about a given movie by title.

  • URL: /movies/:title
  • Method: GET
  • Query Parameters: Title (String)
  • Expected Body Format: N/A
  • Response Format: JSON

Post New Movie

This endpoint posts a new movie.

  • URL: /movies/:title
  • Method: POST
  • Query Parameters: Title (String)
  • Expected Body Format: JSON
  • Response Format: JSON

Update Existing Movie

This endpoint updates an existing move, identified by title.

  • URL: /movies/:title
  • Method: PUT
  • Query Parameters: Title (String)
  • Expected Body Format: JSON
  • Response Format: JSON

Delete Existing Movie

This endpoint deletes an existing move, identified by title.

  • URL: /movies/:title
  • Method: DELETE
  • Query Parameters: Title (String)
  • Expected Body Format: N/A
  • Response Format: JSON

Users Endpoints

Get User Information

This endpoint gets information about a given user, identified by name.

  • URL: /users/:name
  • Method: GET
  • Query Parameters: Name (String)
  • Expected Body Format: N/A
  • Response Format: JSON

Create New User

This endpoint adds a new user.

  • URL: /users/:name
  • Method: POST
  • Query Parameters: Name (String)
  • Expected Body Format: JSON
  • Response Format: JSON

Update User Details

This endpoint updates details about an existing user, identified by name.

  • URL: /users/:name
  • Method: PUT
  • Query Parameters: Name (String)
  • Expected Body Format: JSON
  • Response Format: JSON

Add Movie to a User's Favorites

This endpoint adds a new title to a user's list of favorite movies.

  • URL: /users/:name/:title
  • Method: PUT
  • Query Parameters: Name (The user's name; format: string); Title (The title of the movie to be added to the user's favorites; format: string)
  • Expected Body Format: JSON
  • Response Format: JSON

Delete a User

This endpoint deletes an existing user, identified by name.

  • URL: /users/:name
  • Method: DELETE
  • Query Parameters: Name (String)
  • Expected Body Format: N/A
  • Response Format: JSON
